# Private Alpha Launch Checklist

## Authentication and onboarding
- [ ] Auth flow: sign-in, callback, and session persistence validated.
- [ ] Onboarding completes reliably with age-range policy alignment.

## Emotional QA
- [ ] Internal review queues checked for robotic tone and over-guidance.
- [ ] Emotionally strong responses captured as references.
- [ ] Pacing mismatch and fatigue patterns reviewed.

## Support quality
- [ ] Benchmark scenarios run (grief, burnout, loneliness, school overwhelm, shutdown, relationship pressure, money stress, mixed moments).
- [ ] Optionality and low-pressure language confirmed.
- [ ] Response flow feels conversational without segmented seams.

## Memory continuity
- [ ] Memory persistence failures degrade safely with calm fallback response.
- [ ] Continuity cues avoid overconfident “we know you” tone.
- [ ] Memory actions remain scoped and safe.

## Mobile QA
- [ ] Emotional readability on small screens validated.
- [ ] Scrolling density and fatigue checked.
- [ ] Input + keyboard flow verified.
- [ ] Continuation spacing and CTA placement reviewed.

## Parent/privacy boundaries
- [ ] Private-by-default language present.
- [ ] Non-surveillance tone maintained.
- [ ] Parent/privacy boundaries verified for minors.

## Fallback behavior
- [ ] Orchestration exceptions return calm support response.
- [ ] Persistence failures do not break response delivery.
- [ ] Route/log failures are non-fatal and observable.

## Review tooling and observability
- [ ] Internal review endpoint authorization validated.
- [ ] Feedback endpoint validation verified.
- [ ] Observability traces include QA tags and loop signals.
