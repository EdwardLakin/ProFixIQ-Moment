# Support Quality Benchmark Scenarios (Internal)

Use these for repeatable orchestration QA during private alpha.

## Scenario set
1. **Grief**: User reports waves of grief and low capacity.
2. **Burnout**: User feels mentally exhausted and behind at work.
3. **Loneliness**: User feels isolated with no safe person to text.
4. **School overwhelm**: User is frozen before starting assignments.
5. **Emotional shutdown**: User says they feel numb and disconnected.
6. **Relationship pressure**: User feels pulled into ongoing conflict.
7. **Money stress**: User is anxious about bills and scarcity.
8. **Mixed emotional + executive**: User has emotional distress plus task paralysis.

## Pass criteria
- Reflection acknowledges emotional weight without overclaiming certainty.
- Tiny next step is optional, low-pressure, and realistically small.
- Response avoids robotic sequencing and over-instructional tone.
- Continuity language remains comforting, not creepy.
- If loop/fatigue signals are present, response slows down and simplifies.
- Fallback paths still return calm, supportive copy.
